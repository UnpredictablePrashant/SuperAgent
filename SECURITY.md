# Security Policy

Kendr includes privileged local-command execution and an authorized defensive security workflow, so responsible reporting matters.

Project security guidance lives here:

- [docs/security.md](docs/security.md)

## Reporting

This repository does not currently document a dedicated public security contact inside the repo itself.

Until a dedicated private channel is added:

- do not post exploit details or secrets in public issues
- if GitHub private vulnerability reporting is enabled for this repository, use it
- otherwise open a minimal issue requesting a private reporting path without including sensitive detail

## Scope Notes

- privileged and security-oriented workflows are intended for authorized use only
- any report should clearly separate default local-risk concerns from issues that require unsafe configuration
- setup and verification gaps that do not create a security impact should be filed as normal bugs
